faqs:
  - question: "What kinds of custom rules can I build in DataDocks?"
    answer: "Rules cover the full appointment lifecycle: who can book which doors, allowed time windows by carrier or customer, required documents or fields before submission, capacity by day / shift / door, appointment-type restrictions, approval workflows, and automatic notifications when conditions are met. If it's something your team currently enforces with a checklist or a tribal 'don't book that carrier before 6am,' it can become a rule."
  - question: "Do I need an IT team to set up custom rules?"
    answer: "No. Rules are configured in the admin interface — no code required. For facilities with complex operations, a DataDocks customer success manager helps map existing workflows to rules during onboarding. The rule engine grows with you, so you can start with a few core rules and add more as your operation matures."
  - question: "Can different facilities, docks, or shifts have different rules?"
    answer: "Yes. Rules are scoped — some apply site-wide, others only to a specific facility, dock, door, shift, appointment type, carrier, or customer. Multi-site operators run different rule sets at each location from the same DataDocks account without forcing everything to match the slowest facility."
  - question: "What happens when a rule is broken?"
    answer: "Depends on the rule. Some rules hard-block the action (carriers can't book a door they're not approved for). Others warn your internal team so an override is deliberate rather than accidental — useful for capacity rules where an exception might genuinely be needed. External users (carriers, customers) cannot override rules; only approved internal users can."
  - question: "Can rules trigger approvals before an appointment is confirmed?"
    answer: "Yes. Build approval workflows where certain bookings — new carriers, oversized loads, appointments outside standard hours, loads for specific customers — require sign-off from a named user or team before confirming. This replaces the informal email-for-approval workflow most facilities rely on today."
  - question: "Can I change rules without disrupting current appointments?"
    answer: "Yes. Rule changes apply to new bookings going forward; existing appointments keep their current state unless you choose to re-validate them. You can roll out a new rule on a specific day, pilot it on one dock or shift, then expand once you know it's working."
